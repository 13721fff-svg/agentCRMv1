# AgentCRM

Універсальна CRM + Коннектор платформа для управління бізнесом, клієнтами, маркетингом і аналітикою.

## Особливості

- **Багаторівнева система ролей**: Citizen, Individual, Small Business, Medium Business
- **CRM функціонал**: Управління клієнтами, замовленнями, завданнями
- **Маркетинг**: Створення та управління кампаніями
- **Аналітика**: KPI, дашборди, звіти
- **Командна робота**: Управління командою, ролями, зустрічами
- **Каталог послуг**: Маркетплейс для пошуку та надання послуг
- **Мультимовність**: Українська та англійська мови
- **Адаптивний дизайн**: Працює на всіх пристроях

## Технологічний стек

- **Framework**: React Native + Expo
- **Navigation**: Expo Router
- **State Management**: Zustand
- **UI**: TWRNC (Tailwind CSS for React Native)
- **Database**: Supabase
- **Authentication**: Supabase Auth
- **Локалізація**: i18next
- **Icons**: Lucide React Native

## Структура проєкту

```
app/
  ├── (tabs)/         # Головна навігація з табами
  ├── auth/           # Автентифікація (login, register)
  ├── clients/        # Модуль клієнтів
  ├── orders/         # Модуль замовлень
  ├── meetings/       # Модуль зустрічей
  └── ...             # Інші модулі

components/           # Переюзні компоненти UI
  ├── Button.tsx
  ├── Input.tsx
  ├── Card.tsx
  └── ...

store/                # Zustand stores
  ├── authStore.ts
  ├── clientsStore.ts
  ├── ordersStore.ts
  └── appStore.ts

services/             # API сервіси
  └── authService.ts

lib/                  # Бібліотеки та конфігурація
  ├── tw.ts           # TWRNC конфігурація
  ├── i18n.ts         # Локалізація
  └── supabase.ts     # Supabase клієнт

types/                # TypeScript типи
  └── index.ts

locales/              # Переклади
  ├── uk.json
  └── en.json
```

## Основні модулі

### 1. Автентифікація
- Реєстрація та вхід
- Email/password authentication через Supabase
- Onboarding з вибором ролі

### 2. Головна (Dashboard)
- Персоналізований за роллю користувача
- Швидкі дії
- Статистика та метрики
- Останні замовлення та нагадування

### 3. Каталог (для користувачів)
- Категорії послуг
- Пошук виконавців
- Створення заявок

### 4. Клієнти (для бізнесу)
- База контактів
- Історія взаємодій
- Теги та нотатки
- Рейтинг клієнтів

### 5. Замовлення
- Управління замовленнями
- Статуси та workflow
- Призначення виконавців
- Контроль термінів

### 6. Команда (для малого/середнього бізнесу)
- Управління співробітниками
- Ролі та права доступу
- Розподіл завдань

### 7. Зустрічі
- Планування зустрічей
- Протоколи та нотатки
- Прив'язка до клієнтів

### 8. Кампанії (Маркетинг)
- Створення кампаній
- Push, Email, SMS, Banner
- Таргетинг аудиторії
- Аналітика ефективності

### 9. Аналітика
- Дашборди з метриками
- KPI та цілі
- Звіти та експорт
- Візуалізація даних

### 10. Профіль та налаштування
- Персональні дані
- Мова та тема
- Сповіщення
- Управління підпискою

## База даних

Проєкт використовує Supabase з наступними основними таблицями:

- **users** - Користувачі системи
- **organizations** - Організації/компанії
- **clients** - Клієнти бізнесів
- **orders** - Замовлення
- **categories** - Категорії послуг
- **requests** - Заявки від користувачів
- **quotes** - Пропозиції від виконавців
- **campaigns** - Маркетингові кампанії
- **tasks** - Завдання
- **meetings** - Зустрічі
- **kpis** - Показники ефективності
- **reviews** - Відгуки та рейтинги

Усі таблиці захищені Row Level Security (RLS) політиками.

## Ролі користувачів

### Citizen (Користувач)
- Пошук та замовлення послуг
- Перегляд виконавців
- Створення заявок
- Відгуки та рейтинги

### Individual (Індивідуальний підприємець)
- CRM для управління клієнтами
- Управління замовленнями
- Базовий маркетинг
- Статистика

### Small (Малий бізнес)
- Повний CRM функціонал
- Управління командою (до 10 осіб)
- Розширений маркетинг
- Аналітика та KPI
- Зустрічі та планування

### Medium (Середній бізнес)
- Корпоративний CRM
- Команда до 50 осіб
- Ролі та права доступу
- Повна аналітика
- Кастомні звіти
- Інтеграції

## Додавання нових функцій

### Створення нового екрану

1. Створіть файл в `app/` директорії
2. Експортуйте React компонент
3. Додайте `<Stack.Screen>` в `app/_layout.tsx`
4. Додайте переклади в `locales/uk.json` та `locales/en.json`

### Додавання нового store

1. Створіть файл в `store/` директорії
2. Використовуйте Zustand для створення store
3. Експортуйте хук для використання в компонентах

### Робота з базою даних

1. Створіть нову міграцію через Supabase
2. Додайте типи в `types/index.ts`
3. Створіть сервіс в `services/`
4. Використовуйте RLS політики для безпеки
